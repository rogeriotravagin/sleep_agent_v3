# QBR Playbook - Quarterly Business Review

## O que e QBR

QBR (Quarterly Business Review) e uma reuniao estrategica periodica entre CSM e cliente para revisar resultados, alinhar expectativas e planejar o futuro.

> "QBRs nao sao sobre voce. Sao sobre o sucesso do cliente." - Lincoln Murphy

---

## Objetivos do QBR

### Por que Fazer QBRs

| Objetivo | Beneficio |
|----------|-----------|
| **Demonstrar valor** | Justificar investimento |
| **Alinhar expectativas** | Evitar surpresas |
| **Identificar riscos** | Agir proativamente |
| **Encontrar oportunidades** | Expansion |
| **Fortalecer relacionamento** | Retenção |
| **Coletar feedback** | Melhoria contínua |

### Resultados Esperados

```
Apos um bom QBR:
✓ Cliente ve valor claramente
✓ Objetivos proximos definidos
✓ Riscos identificados e mitigados
✓ Oportunidades mapeadas
✓ Relacionamento fortalecido
✓ Próximos passos claros
```

---

## Tipos de QBR

### Por Frequencia

| Tipo | Frequencia | Tier | Duracao |
|------|------------|------|---------|
| QBR | Trimestral | Enterprise | 60-90 min |
| Semi-annual | Semestral | Mid-Market | 45-60 min |
| Annual | Anual | SMB | 30-45 min |
| EBR (Executive) | Anual | Top accounts | 90-120 min |

### Por Objetivo

| Tipo | Foco | Quando |
|------|------|--------|
| Standard | Revisao regular | Trimestre |
| Renewal | Pre-renovacao | 90 dias antes |
| Recovery | Cliente em risco | Quando necessario |
| Expansion | Upsell focus | Momento certo |

## Preparacao do QBR

### Checklist Pre-QBR (1-2 semanas antes)

```
Coleta de Dados:
[ ] Health Score atualizado
[ ] Metricas de uso (ultimos 90 dias)
[ ] Tickets e issues resolvidos
[ ] NPS/CSAT recentes
[ ] ROI estimado
[ ] Historico de interacoes

Preparacao:
[ ] Deck atualizado com dados
[ ] Casos de sucesso relevantes
[ ] Novidades de produto relevantes
[ ] Oportunidades de expansion mapeadas
[ ] Riscos identificados e plano

Logistica:
[ ] Reuniao agendada e confirmada
[ ] Stakeholders certos convidados
[ ] Deck enviado antecipadamente (opcional)
[ ] Link/sala confirmada
```

### Stakeholders Necessarios

| Lado Cliente | Lado Fornecedor |
|--------------|-----------------|
| Sponsor executivo | CSM |
| Usuario principal | CSM Manager (opcional) |
| IT/Tecnico (se relevante) | Especialista (se necessario) |
| Compras (se pre-renovacao) | Executivo (top accounts) |

## Conteudo do QBR

### 1. Abertura (5 min)

**Objetivo:** Criar conexao e definir tom.

```
Roteiro:
- Agradecer a presenca
- Confirmar agenda
- Definir expectativas
- Check-in informal
```

### 2. Retrospectiva (10 min)

**Objetivo:** Mostrar valor entregue.

```
Estrutura:
1. Objetivos do periodo anterior
2. O que foi alcancado
3. Principais marcos
4. Desafios superados
5. ROI demonstrado
```

**Template de Slide:**

```
┌─────────────────────────────────────────────┐
│     RETROSPECTIVA Q[X] - [CLIENTE]          │
├─────────────────────────────────────────────┤
│                                              │
│  Objetivo: [Meta definida]                  │
│  Resultado: [O que alcancamos]              │
│                                              │
│  Destaques:                                  │
│  • [Conquista 1]                            │
│  • [Conquista 2]                            │
│  • [Conquista 3]                            │
│                                              │
│  Valor Entregue:                            │
│  • Tempo economizado: X horas/mes           │
│  • Receita influenciada: $X                 │
│  • Produtividade: +X%                       │
│                                              │
└─────────────────────────────────────────────┘
```

### 3. Situacao Atual (10 min)

**Objetivo:** Compartilhar dados e saude.

```
Metricas a Apresentar:
- Health Score e tendencia
- Uso (logins, features, usuarios)
- Engajamento (reunioes, respostas)
- Suporte (tickets, satisfacao)
- Comparativo com benchmark
```

**Template de Slide:**

```
┌─────────────────────────────────────────────┐
│       SITUACAO ATUAL - [CLIENTE]            │
├─────────────────────────────────────────────┤
│                                              │

│                                              │
│  Uso Mensal:                                │
│  • Logins: 450 (+15%)                       │
│  • Features: 8/12 (67%)                     │
│  • Usuarios ativos: 25/30 (83%)             │
│                                              │
│  Suporte:                                   │
│  • Tickets: 5 (↓ de 8)                      │
│  • CSAT: 4.5/5                              │
│  • Tempo medio: 4h                          │
│                                              │
│  vs Benchmark: Acima da media               │
│                                              │
└─────────────────────────────────────────────┘
```

### 4. Roadmap e Novidades (10 min)

**Objetivo:** Mostrar evolucao e relevancia.

```
Conteudo:
- Features lancadas (relevantes para cliente)
- Features em desenvolvimento
- Como afeta o cliente
- Oportunidades de aproveitamento
```

### 5. Objetivos Proximo Periodo (10 min)

**Objetivo:** Alinhar metas e expectativas.

```
Perguntas a Responder:
- Quais os objetivos de negocio?
- O que seria sucesso no proximo Q?
- Quais desafios antecipados?
- Onde podemos ajudar mais?
```

**Template:**

```
│  • Metrica: [Como medir]                    │
│  • Meta: [Numero ou %]                      │
│  • Acao: [O que faremos]                    │
│                                              │
│  Objetivo 3: [Descrição]                    │
│  ...                                         │
│                                              │
└─────────────────────────────────────────────┘
```

### 6. Plano de Acao (10 min)

**Objetivo:** Definir proximos passos concretos.

```
Template de Plano:

| Acao | Responsavel | Prazo | Status |
|------|-------------|-------|--------|
| [Acao 1] | [Nome] | [Data] | Pendente |
| [Acao 2] | [Nome] | [Data] | Pendente |
| [Acao 3] | [Nome] | [Data] | Pendente |
```

### 7. Encerramento (5 min)

**Objetivo:** Confirmar alinhamento e coletar feedback.

```
Checklist de Encerramento:
[ ] Resumir principais pontos
[ ] Confirmar proximos passos
[ ] Agendar proximo QBR
[ ] Pedir feedback sobre a reuniao
[ ] Agradecer participacao
```

## QBR de Renovacao

### Diferencial

QBR de renovacao ocorre 60-90 dias antes do vencimento e tem foco adicional na continuidade.

### Elementos Extras

```
1. Revisao completa do periodo contratual
2. ROI total demonstrado
3. Benchmark vs objetivos originais
4. Proposta de renovacao
5. Discussao de termos
6. Oportunidades de expansion
7. Comprometimento
```

### Timeline Pre-Renovacao

```
-90 dias: Iniciar preparacao
-75 dias: QBR de renovacao
-60 dias: Proposta enviada
-45 dias: Follow-up e negociacao
-30 dias: Fechamento
-14 dias: Contrato para assinatura
-7 dias:  Final push
Dia 0:    Renovacao
```

## QBR para Clientes em Risco

### Adaptacoes

| Elemento | QBR Normal | QBR Risco |
|----------|------------|-----------|
| Tom | Celebracao | Resolucao |
| Foco | Futuro | Problemas |
| Stakeholders | Operacionais | + Executivos |
| Duracao | 60 min | 90 min |
| Follow-up | Mensal | Semanal |

### Estrutura Adaptada

```
1. Reconhecer situacao
2. Escutar preocupacoes
3. Diagnosticar raiz do problema
4. Apresentar plano de recuperacao
5. Comprometer recursos
6. Definir checkpoints
7. Agendar follow-up proximo
```

## Boas Praticas

### Do's

1. **Prepare-se** - Dados atualizados e relevantes
2. **Personalize** - Conteudo especifico para cliente
3. **Escute mais** - QBR e dialogo, nao monólogo
4. **Seja honesto** - Sobre problemas e limitacoes
5. **Documente** - Atas e acoes
6. **Siga up** - Cumpra o prometido

### Don'ts

1. **Nao leia slides** - Apresente, nao leia
2. **Nao ignore elefantes** - Aborde problemas
3. **Nao venda demais** - QBR nao e pitch
4. **Nao seja generico** - Personalize sempre
5. **Nao prometa demais** - Seja realista
6. **Nao esqueca follow-up** - Acao e crucial
