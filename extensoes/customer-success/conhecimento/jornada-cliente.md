# Jornada do Cliente

## Visao Geral

A jornada do cliente em CS comeca apos a venda e continua durante todo o ciclo de vida.

---

## Fases da Jornada CS

### Modelo de 6 Fases

```
┌──────────────────────────────────────────────────────────────┐
│                    JORNADA DO CLIENTE CS                      │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  ONBOARDING → ADOPTION → VALUE → GROWTH → RENEWAL → ADVOCACY │
│   (0-30d)     (30-90d)   (90d+)  (Ongoing) (Pre-renov) (Post) │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

---

## Fase 1: Onboarding (Dias 0-30)

### Objetivo
Levar cliente ao primeiro valor rapidamente.

### Marcos Criticos

| Marco | Timing | Indicador |
|-------|--------|-----------|
| Boas-vindas | Dia 0 | Email + call agendada |
| Kickoff | Dia 1-3 | Reuniao realizada |
| Setup | Dia 3-7 | Configuracao completa |
| First Value | Dia 7-14 | Primeiro resultado |
| Treinamento | Dia 14-21 | Time capacitado |
| Health Check | Dia 30 | Avaliacao de sucesso |

### Touchpoints

```
Dia 0:  [Email boas-vindas] [Acesso liberado]
Dia 1:  [Call kickoff agendada]
Dia 3:  [Kickoff realizado] [Setup iniciado]
Dia 7:  [Setup completo] [Primeiro uso]
Dia 14: [Treinamento] [First value]
Dia 21: [Check-in]
Dia 30: [Health check] [Transicao para CSM]
```

### Red Flags

- Nao completou setup em 7 dias
- Sem login apos 3 dias
- Kickoff remarcado 2+ vezes
- Usuario nao comparece a treinamento

## Fase 2: Adoption (Dias 30-90)

### Objetivo
Estabelecer uso regular e workflow.

### Marcos Criticos

| Marco | Timing | Indicador |
|-------|--------|-----------|
| Uso semanal | Dia 30-45 | Login consistente |
| Feature core | Dia 45-60 | Adocao principal |
| Workflow | Dia 60-75 | Processo estabelecido |
| Time engajado | Dia 75-90 | Multiplos usuarios |

### Indicadores de Adocao

```
Nivel 1 - Basico:
- Login semanal
- 1-2 features usadas
- 1 usuario ativo

Nivel 2 - Regular:
- Login diario
- 50%+ features usadas
- 3+ usuarios ativos

Nivel 3 - Power User:
- Multiplos logins/dia
- 80%+ features
- Departamento inteiro
```

### Intervencoes

| Situacao | Acao |
|----------|------|
| Uso baixo | Call de engajamento |
| Feature ignorada | Treinamento focado |
| 1 usuario apenas | Expansao de usuarios |
| Uso declinando | Diagnostico |

---

## Fase 3: Value Realization (90 dias+)

### Objetivo
Garantir que cliente esta alcancando ROI.

### Elementos de Valor

```
VALOR = Resultados Tangiveis + Beneficios Intangiveis

Tangiveis:
- Receita gerada
- Custo reduzido
- Tempo economizado
- Produtividade aumentada

Intangiveis:
- Satisfacao do time
- Confianca
- Reducao de stress
- Competitividade
```

### Demonstrando Valor

| Metodo | Frequencia | Formato |
|--------|------------|---------|
| Dashboard metricas | Sempre visivel | Self-service |
| Email de valor | Mensal | Automatizado |
| QBR | Trimestral | Reuniao |
| Business Case | Anual | Documento |

### Template de Value Statement

```
"Nos ultimos [periodo], voce:
- Economizou [X horas] de trabalho manual
- Aumentou [metrica] em [Y%]
- Reduziu [problema] em [Z%]
- ROI estimado: [valor]"
```

## Fase 4: Growth (Ongoing)

### Objetivo
Expandir uso e receita.

### Oportunidades de Expansion

| Tipo | Descricao | Indicadores |
|------|-----------|-------------|
| Upsell | Plano superior | Alto uso, limites atingidos |
| Cross-sell | Produto adicional | Necessidade adjacente |
| More Users | Mais licencas | Equipe crescendo |
| New Teams | Outros departamentos | Sucesso demonstrado |

### Sinais de Prontidao

```
Pronto para Expansion:
✓ Health Score >80
✓ NPS 9-10
✓ Uso proximo do limite
✓ Pedindo features premium
✓ Indicando para outros

Nao Pronto:
✗ Health Score <60
✗ Tickets abertos
✗ Uso baixo
✗ Stakeholder desengajado
```

---

## Fase 5: Renewal (Pre-Renovacao)

### Objetivo
Garantir continuidade do relacionamento.

### Timeline de Renovacao

```
-90 dias: Iniciar processo de renovacao
-60 dias: QBR de renovacao
-45 dias: Proposta enviada
-30 dias: Negociacao
-14 dias: Contrato para assinatura
-7 dias:  Follow-up final
Dia 0:    Renovacao ou churn
```

### Elementos do QBR de Renovacao

1. **Retrospectiva**
   - Resultados alcancados
   - Valor entregue
   - Momentos importantes

2. **Situacao Atual**
   - Health Score
   - Uso e adocao
   - Feedback coletado

3. **Visao de Futuro**
   - Objetivos proximo periodo
   - Roadmap relevante
   - Oportunidades

4. **Proposta**
   - Termos de renovacao
   - Opcoes de expansion
   - Proximos passos

## Fase 6: Advocacy (Pos-Renovacao)

### Objetivo
Transformar cliente em promotor.

### Niveis de Advocacy

```
Nivel 1: Satisfeito
- Renovaria se perguntado
- NPS 7-8

Nivel 2: Leal
- Defende de criticas
- NPS 9

Nivel 3: Promotor
- Recomenda ativamente
- NPS 10

Nivel 4: Evangelista
- Participa de marketing
- Case study, palestras
```

### Programas de Advocacy

| Programa | Descricao | Beneficio Cliente |
|----------|-----------|-------------------|
| Referencias | Indicar prospects | Desconto/credito |
| Case Study | Historia de sucesso | Exposicao |
| Advisory Board | Conselho de clientes | Influencia |
| Beta Program | Acesso antecipado | Novidades |
| Speaking | Palestras/Webinars | Visibilidade |
