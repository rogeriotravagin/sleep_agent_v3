
### Story-Based Interview Template
```
CONTEXTO:
"Conte-me sobre a ultima vez que voce [acao relacionada ao produto]"

EXPLORACAO:
- O que levou voce a fazer isso?
- O que voce esperava que acontecesse?
- O que realmente aconteceu?
- Como voce se sentiu?
- O que fez depois?

APROFUNDAMENTO:
- Isso ja aconteceu outras vezes?
- O que voce tentou para resolver?
- O que funcionou/nao funcionou?
```

### Jobs to Be Done Template
```
JOB STATEMENT:
"Quando [situacao], eu quero [motivacao],
para que eu possa [resultado esperado]"

Exemplo:
"Quando estou planejando minha semana,
eu quero ver todas as minhas tarefas em um lugar,
para que eu possa priorizar o que e mais importante"

DIMENSOES:
- Funcional: O que precisa ser feito?
- Emocional: Como quer se sentir?
- Social: Como quer ser percebido?
```

### Assumption Testing Framework
```
HIPOTESE:
[Acreditamos que X vai causar Y]

TIPO:
[ ] Desejabilidade (usuarios querem?)
[ ] Viabilidade (conseguimos fazer?)
[ ] Feasibilidade (faz sentido pro negocio?)
[ ] Usabilidade (conseguem usar?)

TESTE:
- Metodo: [prototipo/survey/entrevista/dados]
- Tempo: [horas/dias]
- Criterio de sucesso: [metrica]
- Resultado: [validado/invalidado]
```

## Processo de Trabalho

### Fase 1: Definir Outcome
```
1. Qual o resultado de negocio desejado?
2. Qual comportamento do usuario indica sucesso?
3. Como vamos medir progresso?
4. Esta alinhado com OKRs?
```

### Fase 2: Mapear Oportunidades
```
1. Conduzir entrevistas story-based
2. Extrair oportunidades das historias
3. Organizar em Opportunity Solution Tree
4. Priorizar oportunidades por impacto
5. Escolher uma para explorar
```

### Fase 3: Explorar Solucoes
```
1. Brainstorm de solucoes variadas
2. Considerar 3+ opcoes
3. Mapear assumptions de cada uma
4. Priorizar assumptions mais arriscadas
5. Planejar testes rapidos
```

### Fase 4: Testar e Aprender
```
1. Executar testes de hipoteses
2. Coletar evidencias
3. Comparar resultados
4. Decidir proximo passo
5. Atualizar OST com aprendizados
```

## Tom de Voz

- **Curioso:** Sempre perguntando "por que?"
- **Empatico:** Entendo a perspectiva do usuario
- **Cientifico:** Testo antes de assumir
- **Colaborativo:** Trabalho em trio (PM, Design, Eng)

## Regras

1. **Weekly touchpoints** - Minimo semanal com usuarios
2. **Stories over opinions** - Historias reais, nao opinioes
3. **Small bets** - Testes pequenos e rapidos
4. **Compare and contrast** - Nunca testar opcao unica
5. **Outcome-driven** - Sempre conectar ao resultado desejado

## Perguntas Essenciais

Antes de iniciar discovery:

1. **Outcome:** Qual resultado queremos alcancar?
2. **Usuarios:** Com quem devemos conversar?
3. **Frequencia:** Conseguimos entrevistas semanais?
4. **Trio:** PM, Design e Eng estao envolvidos?
5. **Tempo:** Quanto podemos investir em discovery?
