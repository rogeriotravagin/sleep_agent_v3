# OKRs para Produto

Guia pratico para implementar Objectives and Key Results em times de produto.

## O que sao OKRs

```
OKR = Objective + Key Results

OBJECTIVE: O que queremos alcancar
- Qualitativo e inspirador
- Ambicioso mas alcancavel
- Memoravel

KEY RESULTS: Como sabemos se chegamos la
- Quantitativos e mensuraveis
- Especificos com numeros
- 2-5 por objetivo
```

## Historia e Origem

- **1970s:** Andy Grove cria na Intel
- **1999:** John Doerr leva para Google
- **Hoje:** Usado por Spotify, LinkedIn, Netflix, Airbnb

## Estrutura de OKRs

### Template Basico
```
OBJECTIVE: [Verbo inspirador] + [Resultado desejado]

KR1: [Metrica] de [X atual] para [Y desejado] ate [data]
KR2: [Metrica] de [X atual] para [Y desejado] ate [data]
KR3: [Metrica] de [X atual] para [Y desejado] ate [data]
```

### Exemplo Produto
```
OBJECTIVE: Transformar o onboarding em uma experiencia encantadora

KR1: Aumentar taxa de ativacao de 32% para 50%
KR2: Reduzir tempo ate primeiro valor de 15min para 5min
KR3: Aumentar NPS de novos usuarios de 25 para 45
```

## Niveis de OKRs

```
         EMPRESA
    (Anuais/Semestrais)
            |
            v
        AREA/BU

            v

     (Trimestrais)

    (Opcional/Trimestral)
```

### Alinhamento
```
Company OKR: Tornar-nos lider de mercado

    Product OKR: Criar produto que usuarios amam
        |
        +-> Time A OKR: Melhorar retencao
        +-> Time B OKR: Expandir para novo segmento
        +-> Time C OKR: Reduzir fricao de pagamento
```

## Principios Essenciais

### 1. Menos e Mais
```
ERRADO:
- 10 objetivos com 5 KRs cada
- Time nao lembra nenhum
- Foco disperso

CERTO:
- 3-5 objetivos
- 2-4 KRs por objetivo
- Time sabe de cor
```

### 2. Outcomes, nao Outputs
```
ERRADO (Output):
KR: Lancar feature X ate dia 15

CERTO (Outcome):
KR: Aumentar conversao de trial em 20% com nova feature
```

### 3. Ambitious mas Achievable
```
Regra do 70%:
- Se atingir 100% sempre = muito facil
- Se atingir 50% sempre = muito dificil
- Alvo: Atingir ~70% e considerado sucesso

Moonshots (Google):
- Objetivos ambiciosos
- 60-70% e vitoria
- Falhar com 50% ainda e progresso
```

### 4. Transparencia Total
```
- Todos os OKRs sao publicos
- Qualquer um pode ver OKRs de qualquer time
- Permite alinhamento organico
- Evita duplicacao de esforcos
```

## Cadencia

### Ciclo Trimestral
```
SEMANA -2 a -1: PLANEJAMENTO
[ ] Revisar OKRs do trimestre anterior
[ ] Analisar o que funcionou/nao funcionou
[ ] Rascunhar novos OKRs
[ ] Alinhar com stakeholders

SEMANA 1: KICK-OFF
[ ] Finalizar OKRs
[ ] Comunicar para toda empresa
[ ] Garantir entendimento

SEMANAS 2-12: EXECUCAO
[ ] Check-ins semanais (15min)
[ ] Atualizar progresso
[ ] Identificar blockers
[ ] Ajustar taticas se necessario

SEMANA 13: RETROSPECTIVA
[ ] Scoring final
[ ] Learnings documentados
[ ] Input para proximo ciclo
```

### Check-in Semanal
```
Para cada OKR:
1. Progresso atual: X%
2. Confianca de atingir: Alta/Media/Baixa
3. O que fizemos essa semana?
4. O que faremos proxima semana?
5. Algum blocker?
```

## Scoring

### Escala 0-1.0
```
1.0 = Atingiu 100% ou mais
0.7 = Atingiu ~70% (sucesso!)
0.5 = Progresso significativo
0.3 = Algum progresso
0.0 = Nenhum progresso

Exemplo:
KR: Aumentar NPS de 30 para 50
- Resultado: NPS = 42
- Score: (42-30)/(50-30) = 0.6
```

### Interpretacao
```
Score 0.7-1.0: Sucesso, talvez muito facil
Score 0.4-0.6: Bom esforco, aprendizado
Score 0.0-0.3: Algo deu errado, investigar

Nota: OKRs NAO sao avaliacao de performance!
      Sao ferramentas de alinhamento e foco
```

## OKRs de Produto - Exemplos

### Exemplo 1: Foco em Retencao
```
OBJECTIVE: Criar um produto que usuarios nao conseguem largar

KR1: Aumentar D30 retention de 25% para 40%
KR2: Aumentar DAU/MAU ratio de 15% para 30%
KR3: Reduzir churn rate de 8% para 4% ao mes
```

### Exemplo 2: Foco em Crescimento
```
OBJECTIVE: Acelerar crescimento organico atraves de produto excepcional

KR1: Aumentar NPS de 35 para 55
KR2: Atingir K-factor de 1.2 (viral coefficient)
KR3: Aumentar % de usuarios vindos de referral de 15% para 35%
```

### Exemplo 3: Foco em Ativacao
```
OBJECTIVE: Garantir que todo novo usuario experimente o valor do produto

KR1: Aumentar taxa de ativacao de 40% para 65%
KR2: Reduzir time-to-value de 3 dias para 1 dia
KR3: Aumentar % usuarios que completam onboarding de 50% para 80%
```

### Exemplo 4: Foco em Monetizacao
```
OBJECTIVE: Construir modelo de monetizacao sustentavel e escalavel

KR1: Aumentar conversao trial->pago de 8% para 15%
KR2: Aumentar ARPU de $50 para $75
KR3: Atingir Net Revenue Retention de 110%
```

## Erros Comuns

### 1. KRs como Tarefas
```
ERRADO:
KR: Implementar sistema de notificacoes

CERTO:
KR: Aumentar re-engajamento via notificacoes em 25%
```

### 2. Objectives Vagos
```
ERRADO:
O: Melhorar o produto

CERTO:
O: Tornar o onboarding tao simples que qualquer um consegue comecar em 5 minutos
```

### 3. KRs Nao Mensuraveis
```
ERRADO:
KR: Melhorar a experiencia do usuario

CERTO:
KR: Aumentar CSAT de 3.5 para 4.5
```

### 4. Muitos OKRs
```
ERRADO:
8 objetivos com 6 KRs cada = 48 coisas para acompanhar

CERTO:
3 objetivos com 3 KRs cada = 9 coisas focadas
```

### 5. Nao Conectar a Estrategia
```
ERRADO:
OKRs de time desconectados da visao da empresa

CERTO:
Cada OKR de time contribui para OKR da empresa
```

## OKRs vs Outras Ferramentas

```
| Aspecto | OKRs | KPIs | Metas Tradicionais |
|---------|------|------|-------------------|
| Frequencia | Trimestral | Continuo | Anual |
| Ambicao | Stretch (70%) | 100% | 100% |
| Transparencia | Publica | Variavel | Restrita |
| Vinculo bonus | Nao | Sim | Sim |
| Foco | Outcomes | Health metrics | Outputs |
```

## Templates

### OKR Canvas
```markdown
# OKR [Trimestre] - [Time]

## Contexto
[Por que esses objetivos? Qual a estrategia?]

---

## Objetivo 1: [Titulo inspirador]

**Por que isso importa:**
[Conexao com estrategia]

**Key Results:**

| KR | Baseline | Target | Atual | Score |
|----|----------|--------|-------|-------|
| [KR1] | [X] | [Y] | [Z] | [0-1] |
| [KR2] | [X] | [Y] | [Z] | [0-1] |
| [KR3] | [X] | [Y] | [Z] | [0-1] |

**Iniciativas planejadas:**
- [Iniciativa 1]
- [Iniciativa 2]

---

## Objetivo 2: [...]
[Mesmo formato]

---

## Riscos e Dependencias
- [Risco 1]
- [Dependencia de time X]
```

### Check-in Semanal
```markdown
# Check-in OKR - Semana [N]

## Resumo
[Status geral: On track / At risk / Off track]

## Por Objetivo

### O1: [Nome]
- **Progresso:** X%
- **Confianca:** [Alta/Media/Baixa]
- **Highlights:** [O que avancou]
- **Blockers:** [O que esta travando]
- **Proximos passos:** [O que vamos fazer]

[Repetir para cada objetivo]

## Pedir Ajuda
[Algo que precisamos de outros times?]
```
