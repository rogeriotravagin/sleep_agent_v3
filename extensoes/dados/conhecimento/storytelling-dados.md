# Storytelling com Dados - Base de Conhecimento

## O que e Storytelling com Dados

Storytelling com dados e a habilidade de transformar numeros e analises em narrativas que geram compreensao e acao. Nao basta ter os dados certos - e preciso comunica-los de forma que a audiencia entenda, se importe e aja.

> "O maior inimigo da comunicacao e a ilusao de que ela aconteceu." - George Bernard Shaw

---

## Os 3 Elementos Essenciais

### 1. Dados (O que)
- Fatos, numeros, evidencias
- A materia-prima da historia
- Deve ser preciso e confiavel

### 2. Narrativa (Por que)
- O contexto e significado
- A conexao emocional
- O "so what?" respondido

### 3. Visual (Como)
- A forma de apresentar
- Clareza e simplicidade
- Foco na mensagem

```
     [DADOS]
        |
        v
   +----------+
   | INSIGHT  | <- A interseccao e onde a magica acontece
   +----------+
      /    \
     v      v
[NARRATIVA] [VISUAL]
```

## Framework SBG (Situation, Behavior, Goal)

### Estrutura Narrativa

```
1. SITUACAO (Onde estamos)
   - Contexto atual
   - Dados de baseline
   - Por que isso importa

2. COMPLICACAO (O problema/oportunidade)
   - O que mudou
   - Tendencia preocupante ou oportunidade
   - Impacto se nada for feito

3. RESOLUCAO (O que fazer)
   - Recomendacao clara
   - Dados que suportam
   - Proximos passos concretos
```

### Exemplo:

**Situacao:** "Nossa taxa de retencao historica e de 75% ao mes."

**Complicacao:** "Nos ultimos 3 meses, caiu para 65%, representando R$ 500K em receita perdida."

**Resolucao:** "Identificamos que 80% do churn vem de usuarios que nao completaram o onboarding. Proponho reformular o onboarding, projetando recuperar 5pp de retencao."

## Framework So What (Avinash Kaushik)

Todo dado apresentado deve passar por este teste:

### 1. So What? (E dai?)
- Por que esse numero importa?
- Qual a implicacao para o negocio?
- Se nao muda nenhuma decisao, nao apresente.

### 2. What Changed? (O que mudou?)
- Comparar com periodo anterior
- Comparar com meta
- Comparar com benchmark de mercado
- Numero isolado nao significa nada

### 3. Why? (Por que?)
- Qual a causa raiz?
- O que explica a mudanca?
- Evidencia vs. hipotese

### 4. Now What? (E agora?)
- Acao clara e especifica
- Quem e responsavel?
- Qual o prazo?

**Exemplo ruim:**
"Tivemos 50.000 visitantes no site."

**Exemplo bom:**
"Tivemos 50.000 visitantes, 20% acima do mes anterior, impulsionado pela campanha de Black Friday. Sugiro replicar essa abordagem no Natal, com lancamento em 1 semana."

## Piramide de Barbara Minto

Estrutura para comunicacao executiva.

```
         [CONCLUSAO PRINCIPAL]
                  |
    +-------------+-------------+
    |             |             |
[Argumento 1] [Argumento 2] [Argumento 3]
    |             |             |
[Evidencia]   [Evidencia]   [Evidencia]
```

### Regras:
1. **Comece pela conclusao** - Executivos querem o resultado primeiro
2. **Agrupe argumentos** - Maximo 3-5 pontos principais (regra MECE)
3. **Suporte com evidencias** - Dados que provam cada argumento
4. **Ordem logica** - Cronologica, por importancia ou estrutural

### Exemplo:
**Conclusao:** "Devemos investir R$ 200K em automacao de marketing."

**Argumento 1:** "Reduzira custo de aquisicao em 30%" (evidencia: benchmark do setor)

**Argumento 2:** "Liberara 40h/mes do time de marketing" (evidencia: analise de tarefas)

**Argumento 3:** "Payback em 6 meses" (evidencia: projecao financeira)

## Principios de Visualizacao

### 1. Remova o Desnecessario (Data-Ink Ratio)

**Remover:**
- Linhas de grade desnecessarias
- Bordas e caixas
- Efeitos 3D
- Sombras e gradientes
- Cores que nao comunicam
- Legendas que podem ser rotulos diretos

### 2. Foco e Atencao

Use elementos visuais para guiar o olhar:
- **Cor:** Destaque o que importa
- **Tamanho:** Maior = mais importante
- **Posicao:** Canto superior esquerdo primeiro
- **Contraste:** Claro vs. escuro

### 3. Escolha o Grafico Certo

| Para mostrar... | Use... |
|-----------------|--------|
| Comparacao entre categorias | Barras |
| Tendencia no tempo | Linha |
| Partes de um todo (max 5) | Pizza |
| Distribuicao | Histograma |
| Correlacao | Scatter |
| Fluxo/jornada | Sankey |

### 4. Titulo que Conta a Historia

**Ruim:** "Vendas por Mes"

**Bom:** "Vendas cresceram 25% apos campanha de Marco"

O titulo deve ser o insight, nao a descricao do grafico.

## Estrutura de Apresentacao

### Para Executivos (5-10 min)

```
1. Contexto rapido (30 seg)
   - Por que estamos aqui

2. Insight principal (1 min)
   - A conclusao mais importante

3. Evidencias (3-5 min)
   - 2-3 dados que suportam
   - Visualizacoes simples

4. Recomendacao (1 min)
   - O que fazer
   - Decisao necessaria

5. Discussao
   - Perguntas e proximos passos
```

### Para Times Operacionais (30-45 min)

```
1. Contexto e objetivos (5 min)
   - O que analisamos e por que

2. Metodologia (5 min)
   - Como fizemos
   - Limitacoes

3. Descobertas detalhadas (15-20 min)
   - Dados principais
   - Analises de suporte
   - Visualizacoes

4. Implicacoes (5 min)
   - O que isso significa

5. Recomendacoes (5 min)
   - Acoes sugeridas
   - Proximos passos

6. Discussao (5-10 min)
```

## Erros Comuns a Evitar

### 1. Excesso de Dados
- Mostrar tudo que foi analisado
- Graficos demais em uma tela
- **Solucao:** Menos e mais. Escolha os 3-5 insights mais importantes.

### 2. Falta de Contexto
- Numeros sem comparacao
- Nao explicar por que importa
- **Solucao:** Sempre compare e explique "so what".

### 3. Visualizacao Errada
- Pizza com 10 fatias
- Grafico de linha para categorias
- **Solucao:** Pergunte "o que quero mostrar?" antes de escolher o grafico.

### 4. Conclusao Enterrada
- Comecar pela metodologia
- Deixar o insight para o final
- **Solucao:** Piramide invertida - conclusao primeiro.

### 5. Sem Acao Clara
- Apresentar dados sem recomendacao
- Deixar a decisao em aberto
- **Solucao:** Sempre termine com "recomendo que..." ou "sugiro que...".

## Templates de Narrativa

### Para Resultado Positivo:
```
"[Metrica] cresceu [X%] em [periodo], superando a meta de [Y%].
Esse resultado foi impulsionado por [causa principal].
Para manter esse momentum, sugiro [acao]."
```

### Para Resultado Negativo:
```
"[Metrica] caiu [X%] em [periodo], ficando [Y%] abaixo da meta.
A analise indica que [causa raiz].
Para reverter, proponho [acao 1] e [acao 2], com potencial de recuperar [impacto esperado]."
```

### Para Descoberta/Insight:
```
"Ao analisar [dados], descobrimos que [insight].
Isso significa que [implicacao para o negocio].
Com base nisso, recomendo [acao]."
```

### Para Comparativo:
```
"Comparando [A] com [B], observamos que [diferenca principal].
[A] supera [B] em [aspecto], enquanto [B] e melhor em [outro aspecto].
Dado nosso objetivo de [X], sugiro [escolha] porque [razao]."
```

## Checklist de Qualidade

Antes de apresentar, verifique:

```
[ ] Comeco pela conclusao principal?
[ ] Todo numero tem comparacao (antes, meta, benchmark)?
[ ] Cada dado responde "so what"?
[ ] Visualizacoes sao simples e claras?
[ ] Titulos contam a historia (insight, nao descricao)?
[ ] Removi elementos visuais desnecessarios?
[ ] Recomendacao esta clara e acionavel?
[ ] Adapotei linguagem para minha audiencia?
[ ] Preparei respostas para perguntas provaveis?
```
