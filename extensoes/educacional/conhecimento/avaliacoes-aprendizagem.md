# Avaliações de Aprendizagem - Base de Conhecimento

## Propósito da Avaliação

Avaliação não é apenas medir. É parte integrante do processo de aprendizagem.

> "A avaliação deve servir para melhorar a aprendizagem, não apenas para medir." — Dylan Wiliam

## Tipos de Avaliação

### 1. Avaliação Diagnóstica

**Quando:** Início do curso/módulo

**Para quê:**
- Medir conhecimento prévio
- Identificar lacunas
- Personalizar conteúdo
- Formar grupos

**Formatos:**
- Quiz inicial
- Autoavaliação
- Mapa de conhecimento
- Entrevista

**Não é para nota. É para direcionar.**

---

### 2. Avaliação Formativa

**Quando:** Durante o processo

**Para quê:**
- Acompanhar progresso
- Dar feedback imediato
- Identificar dificuldades
- Ajustar instrução

**Formatos:**
- Quizzes rápidos
- Exercícios de prática
- Check-ins
- Reflexões
- Observação

**Características:**
- Baixo risco (pouco peso na nota)
- Feedback rico
- Oportunidade de melhoria
- Frequente

---

### 3. Avaliação Somativa

**Quando:** Final do módulo/curso

**Para quê:**
- Certificar aprendizagem
- Verificar atingimento de objetivos
- Atribuir nota/certificação
- Comparar com padrões

**Formatos:**
- Prova final
- Projeto integrador
- Portfólio
- Apresentação
- Certificação

**Características:**
- Alto risco (peso na nota)
- Abrangente
- Padronizada
- Menos frequente

---

## Comparativo

| Aspecto | Diagnóstica | Formativa | Somativa |
|---------|-------------|-----------|----------|
| Quando | Início | Durante | Final |
| Propósito | Conhecer | Melhorar | Certificar |
| Frequência | Uma vez | Contínua | Pontual |
| Feedback | Imediato | Contínuo | Final |
| Peso | Nenhum | Baixo | Alto |
| Foco | Entrada | Processo | Saída |

## Formatos de Avaliação

### Questões Objetivas

#### Múltipla Escolha

**Estrutura:**
```
ENUNCIADO: Pergunta clara e completa

a) Alternativa correta
b) Distrator plausível (erro comum 1)
c) Distrator plausível (erro comum 2)
d) Distrator plausível (erro comum 3)
```

**Boas práticas:**
- Enunciado autossuficiente
- Uma única resposta correta
- Distratores plausíveis
- Evitar "todas as anteriores"
- Alternativas de tamanho similar

**Exemplo bom:**
```
Segundo a Taxonomia de Bloom, qual nível cognitivo exige
que o aluno produza algo original?

a) Aplicar
b) Analisar
c) Avaliar
d) Criar ✓
```

**Exemplo ruim:**
```
Bloom é:

a) Um autor
b) Uma taxonomia
c) Um framework
d) Todas as anteriores
```

---

#### Verdadeiro ou Falso

**Estrutura:**
```
Afirmação: [Declaração clara]
( ) Verdadeiro  ( ) Falso

Justificativa (se falso): [Por quê]
```

**Boas práticas:**
- Afirmações inequívocas
- Evitar dupla negação
- Uma ideia por item
- Pedir justificativa

---

#### Associação/Correspondência

**Estrutura:**
```
Coluna A          Coluna B
1. Item 1         ( ) Descrição A
2. Item 2         ( ) Descrição B
3. Item 3         ( ) Descrição C
                  ( ) Descrição D (distrator)
```

**Boas práticas:**
- Listas homogêneas
- Incluir distratores
- Instruções claras

### Questões Dissertativas

#### Resposta Curta

```
Pergunta: Explique em 2-3 frases o conceito de [X].

Critérios:
- Precisão conceitual (2 pts)
- Clareza da explicação (1 pt)
- Exemplo relevante (1 pt)
```

#### Resposta Longa

```
Pergunta: Analise [situação] considerando [aspectos].

Critérios (Rubrica):
- Análise completa dos aspectos (4 pts)
- Fundamentação teórica (3 pts)
- Argumentação lógica (2 pts)
- Conclusão coerente (1 pt)
```

## Rubricas de Avaliação

### O que é Rubrica

Ferramenta que descreve critérios e níveis de desempenho para avaliação.

### Tipos

#### Rubrica Holística

Avalia o trabalho como um todo.

```
| Nível | Descrição |
|-------|-----------|
| 4 - Excelente | Demonstra domínio completo... |
| 3 - Proficiente | Demonstra bom entendimento... |
| 2 - Em desenvolvimento | Demonstra entendimento básico... |
| 1 - Iniciante | Demonstra dificuldade em... |
```

#### Rubrica Analítica

Avalia cada critério separadamente.

```
| Critério | 4 Excelente | 3 Bom | 2 Satisfatório | 1 Insuficiente |
|----------|-------------|-------|----------------|----------------|
| Conteúdo | [descrição] | [descrição] | [descrição] | [descrição] |
| Organização | [descrição] | [descrição] | [descrição] | [descrição] |
| Clareza | [descrição] | [descrição] | [descrição] | [descrição] |
```

### Template de Rubrica

```markdown
## Rubrica: [Nome da Atividade]

### Critério 1: [Nome]
| Nível | Pontos | Descrição |
|-------|--------|-----------|
| Excelente | 4 | [O que caracteriza excelência] |
| Bom | 3 | [O que caracteriza bom desempenho] |
| Satisfatório | 2 | [O que caracteriza desempenho mínimo] |
| Insuficiente | 1 | [O que caracteriza insuficiência] |

### Critério 2: [Nome]
...

### Pontuação Total
- Excelente: 16-20 pontos
- Bom: 12-15 pontos
- Satisfatório: 8-11 pontos
- Insuficiente: < 8 pontos
```

## Feedback Eficaz

### Características

| Bom Feedback | Feedback Ineficaz |
|--------------|-------------------|
| Específico | Vago |
| Acionável | Abstrato |
| Oportuno | Atrasado |
| Focado na tarefa | Focado na pessoa |
| Equilibrado | Só negativo ou só positivo |

### Estrutura do Feedback

```
1. O QUE ESTÁ BOM
   [Pontos positivos específicos]

2. O QUE PODE MELHORAR
   [Aspectos a desenvolver]

3. COMO MELHORAR
   [Sugestões concretas]

4. PRÓXIMOS PASSOS
   [Ações específicas]
```

### Exemplos

**❌ Feedback ruim:**
> "Bom trabalho, mas precisa melhorar."

**✅ Feedback bom:**
> "Sua análise do problema está bem fundamentada, especialmente quando você conectou com a teoria de Bloom. Para melhorar, sugiro incluir um exemplo prático que ilustre sua conclusão. Tente usar um caso real do seu contexto profissional."

## Alinhamento Construtivo

### Conceito (John Biggs)

```
OBJETIVOS ←→ ATIVIDADES ←→ AVALIAÇÕES

Todos devem estar alinhados!
```

### Verificação

| Objetivo (Bloom) | Atividade | Avaliação |
|------------------|-----------|-----------|
| Lembrar | Flashcards | Quiz múltipla escolha |
| Entender | Resumo, discussão | Explicação escrita |
| Aplicar | Exercício prático | Demonstração |
| Analisar | Estudo de caso | Relatório de análise |
| Avaliar | Debate, crítica | Parecer fundamentado |
| Criar | Projeto | Produto + Apresentação |

## Checklist de Avaliação

### Planejamento
- [ ] Objetivos de aprendizagem claros
- [ ] Formato adequado ao objetivo
- [ ] Alinhamento objetivo-atividade-avaliação
- [ ] Rubrica definida
- [ ] Tempo adequado

### Construção
- [ ] Instruções claras
- [ ] Critérios conhecidos pelos alunos
- [ ] Nível de dificuldade apropriado
- [ ] Acessibilidade considerada
- [ ] Revisão feita

### Aplicação
- [ ] Ambiente adequado
- [ ] Suporte disponível
- [ ] Tempo suficiente
- [ ] Regras claras

### Correção
- [ ] Rubrica seguida consistentemente
- [ ] Feedback específico
- [ ] Prazo de retorno cumprido
- [ ] Oportunidade de melhoria

### Análise
- [ ] Resultados compilados
- [ ] Itens problemáticos identificados
- [ ] Ajustes para próxima vez
- [ ] Feedback do processo

    - "taxonomia-bloom.md"

  tags: ["bloom", "objetivos", "verbos"]

    taxa_acerto: null
    indice_discriminacao: null
