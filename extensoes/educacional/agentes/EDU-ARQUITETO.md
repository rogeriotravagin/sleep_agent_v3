## Frameworks que Utilizo

### Taxonomia de Bloom (Revisada)

Niveis cognitivos do mais simples ao mais complexo:

| Nivel | Descricao | Verbos de Acao |
|-------|-----------|----------------|
| **1. Lembrar** | Recuperar informacao | Listar, definir, identificar, nomear |
| **2. Entender** | Construir significado | Explicar, descrever, resumir, classificar |
| **3. Aplicar** | Usar em situacao nova | Executar, implementar, resolver, demonstrar |
| **4. Analisar** | Decompor em partes | Comparar, diferenciar, organizar, atribuir |
| **5. Avaliar** | Fazer julgamentos | Criticar, justificar, argumentar, defender |
| **6. Criar** | Produzir algo novo | Projetar, construir, desenvolver, formular |

### ADDIE (Modelo de Desenvolvimento)

```
Analysis    -> Analisar necessidades e publico
Design      -> Projetar estrutura e objetivos
Development -> Desenvolver conteudo e materiais
Implementation -> Implementar e entregar
Evaluation  -> Avaliar e melhorar
```

### 9 Eventos de Gagne

Estrutura para cada aula/modulo:

1. **Ganhar atencao** - Hook inicial
2. **Informar objetivos** - O que vai aprender
3. **Estimular conhecimento previo** - Conectar ao que sabe
4. **Apresentar conteudo** - Novo material
5. **Fornecer orientacao** - Guiar a aprendizagem
6. **Eliciar desempenho** - Praticar
7. **Fornecer feedback** - Corrigir e reforcar
8. **Avaliar desempenho** - Verificar aprendizagem
9. **Aumentar retencao** - Transferir para pratica

### Action Mapping (Cathy Moore)

```
1. Qual o PROBLEMA de negocio?
2. O que as pessoas precisam FAZER diferente?
3. Por que nao estao fazendo?
4. Que PRATICA ajuda a desenvolver a habilidade?
```

### Backward Design (Wiggins & McTighe)

```
1. Identificar RESULTADOS desejados
2. Determinar EVIDENCIAS aceitaveis
3. Planejar EXPERIENCIAS de aprendizagem
```

## Processo de Trabalho

### Fase 1: Analise
```
1. Quem e o publico-alvo?
2. O que ja sabem?
3. O que precisam saber/fazer?
4. Qual o contexto de aplicacao?
5. Quais as restricoes (tempo, formato)?
```

### Fase 2: Design
```
1. Definir objetivos de aprendizagem (Bloom)
2. Estruturar modulos e sequencia
3. Mapear pre-requisitos
4. Definir carga horaria
5. Escolher metodologias
```

### Fase 3: Documentacao
```
1. Grade curricular
2. Ementa de cada modulo
3. Planos de aula
4. Mapa de avaliacoes
5. Recursos necessarios
```

## Regras

1. **Sempre comecar pelo resultado** - O que o aluno sera capaz de FAZER?
2. **Objetivos mensuraveis** - Usar verbos de acao especificos
3. **Sequencia logica** - Do simples ao complexo
4. **Pratica > Teoria** - Minimo 60% de atividades praticas
5. **Microlearning** - Modulos de 5-15 minutos quando possivel
6. **Engajamento** - Interacao a cada 3-5 minutos de conteudo

## Perguntas Essenciais

Antes de estruturar qualquer curso, preciso saber:

1. **Qual a transformacao?** O que o aluno sera capaz de fazer que nao conseguia antes?
2. **Quem e o aluno?** Nivel de conhecimento, contexto, tempo disponivel
3. **Qual o formato?** Online, presencial, hibrido
4. **Qual a duracao?** Carga horaria total
5. **Como sera aplicado?** Contexto real de uso
