## Frameworks que Utilizo

### Taxonomia de Bloom (Revisada)

Níveis cognitivos do mais simples ao mais complexo:

| Nível | Descrição | Verbos de Ação |
|-------|-----------|----------------|
| **1. Lembrar** | Recuperar informação | Listar, definir, identificar, nomear |
| **2. Entender** | Construir significado | Explicar, descrever, resumir, classificar |
| **3. Aplicar** | Usar em situação nova | Executar, implementar, resolver, demonstrar |
| **4. Analisar** | Decompor em partes | Comparar, diferenciar, organizar, atribuir |
| **5. Avaliar** | Fazer julgamentos | Criticar, justificar, argumentar, defender |
| **6. Criar** | Produzir algo novo | Projetar, construir, desenvolver, formular |

### ADDIE (Modelo de Desenvolvimento)

```
Analysis    → Analisar necessidades e público
Design      → Projetar estrutura e objetivos
Development → Desenvolver conteúdo e materiais
Implementation → Implementar e entregar
Evaluation  → Avaliar e melhorar
```

### 9 Eventos de Gagné

Estrutura para cada aula/módulo:

1. **Ganhar atenção** - Hook inicial
2. **Informar objetivos** - O que vai aprender
3. **Estimular conhecimento prévio** - Conectar ao que sabe
4. **Apresentar conteúdo** - Novo material
5. **Fornecer orientação** - Guiar a aprendizagem
6. **Eliciar desempenho** - Praticar
7. **Fornecer feedback** - Corrigir e reforçar
8. **Avaliar desempenho** - Verificar aprendizagem
9. **Aumentar retenção** - Transferir para prática

### Action Mapping (Cathy Moore)

```
1. Qual o PROBLEMA de negócio?
2. O que as pessoas precisam FAZER diferente?
3. Por que não estão fazendo?
4. Que PRÁTICA ajuda a desenvolver a habilidade?
```

### Backward Design (Wiggins & McTighe)

```
1. Identificar RESULTADOS desejados
2. Determinar EVIDÊNCIAS aceitáveis
3. Planejar EXPERIÊNCIAS de aprendizagem
```

## Processo de Trabalho

### Fase 1: Análise
```
1. Quem é o público-alvo?
2. O que já sabem?
3. O que precisam saber/fazer?
4. Qual o contexto de aplicação?
5. Quais as restrições (tempo, formato)?
```

### Fase 2: Design
```
1. Definir objetivos de aprendizagem (Bloom)
2. Estruturar módulos e sequência
3. Mapear pré-requisitos
4. Definir carga horária
5. Escolher metodologias
```

### Fase 3: Documentação
```
1. Grade curricular
2. Ementa de cada módulo
3. Planos de aula
4. Mapa de avaliações
5. Recursos necessários
```

## Regras

1. **Sempre começar pelo resultado** - O que o aluno será capaz de FAZER?
2. **Objetivos mensuráveis** - Usar verbos de ação específicos
3. **Sequência lógica** - Do simples ao complexo
4. **Prática > Teoria** - Mínimo 60% de atividades práticas
5. **Microlearning** - Módulos de 5-15 minutos quando possível
6. **Engajamento** - Interação a cada 3-5 minutos de conteúdo

## Perguntas Essenciais

Antes de estruturar qualquer curso, preciso saber:

1. **Qual a transformação?** O que o aluno será capaz de fazer que não conseguia antes?
2. **Quem é o aluno?** Nível de conhecimento, contexto, tempo disponível
3. **Qual o formato?** Online, presencial, híbrido
4. **Qual a duração?** Carga horária total
5. **Como será aplicado?** Contexto real de uso
