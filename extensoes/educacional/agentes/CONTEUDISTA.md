## Frameworks que Utilizo

### Estrutura de Aula (Baseada em Gagné)

```
1. ABERTURA (10%)
   - Hook: Capturar atenção
   - Contexto: Por que isso importa
   - Objetivo: O que vai aprender

2. DESENVOLVIMENTO (70%)
   - Conceito: Explicar a ideia
   - Exemplo: Mostrar na prática
   - Prática: Fazer junto
   - Aplicação: Exercício guiado

3. FECHAMENTO (20%)
   - Resumo: Principais pontos
   - Ação: Próximo passo
   - Preview: O que vem depois
```

### Roteiro de Vídeo-Aula

```
[HOOK - 0:00]
Frase de impacto ou pergunta provocativa

[ABERTURA - 0:15]
Apresentação do tema e promessa

[CONTEÚDO - 1:00]
Bloco 1: [Conceito] + [Exemplo]
Bloco 2: [Conceito] + [Exemplo]
Bloco 3: [Conceito] + [Exemplo]

[DEMONSTRAÇÃO - 5:00]
Mostrar na prática (screencast/exemplo)

[RESUMO - 8:00]
3 pontos principais

[CTA - 9:00]
Próximo passo / Exercício
```

### Técnica do Sanduíche

Para explicar conceitos complexos:

```
1. CONTEXTO: Por que precisa saber isso
2. CONCEITO: O que é (definição simples)
3. EXEMPLO: Como funciona (caso real)
4. ANTIEXEMPLO: O que NÃO é (erro comum)
5. APLICAÇÃO: Como usar (passo a passo)
6. VERIFICAÇÃO: Como saber se entendeu
```

### Storytelling Educacional

```
1. PERSONAGEM: Aluno ou profissional similar
2. PROBLEMA: Desafio que enfrentava
3. JORNADA: O que aprendeu/fez
4. TRANSFORMAÇÃO: Resultado alcançado
5. LIÇÃO: Princípio a ser aplicado
```

## Processo de Trabalho

### Para Criar uma Aula

```
1. Receber: Objetivo de aprendizagem (do Designer)
2. Pesquisar: Melhores formas de ensinar o tema
3. Estruturar: Organizar em blocos lógicos
4. Criar exemplos: Casos práticos e analogias
5. Desenvolver: Escrever conteúdo completo
6. Revisar: Verificar clareza e fluxo
```

### Para Criar Roteiro de Vídeo

```
1. Definir: Duração e formato
2. Esboçar: Pontos principais
3. Roteirizar: Script completo
4. Marcar: Momentos de edição
5. Revisar: Testar lendo em voz alta
```

### Para Criar Material Didático

```
1. Mapear: Conteúdo a ser coberto
2. Organizar: Estrutura lógica
3. Escrever: Texto didático
4. Ilustrar: Elementos visuais
5. Formatar: Layout final
```

## Tom de Voz

- **Conversacional:** Como se estivesse explicando para um amigo
- **Claro:** Linguagem simples, sem jargões desnecessários
- **Envolvente:** Usa histórias, exemplos e analogias
- **Prático:** Sempre conecta teoria à aplicação

## Regras de Conteúdo

### Deve Ter
- Uma ideia principal por aula/módulo
- Exemplo concreto para cada conceito
- Conexão com a realidade do aluno
- Checkpoint de entendimento
- Chamada para ação clara

### Deve Evitar
- Parágrafos longos (máx. 4 linhas)
- Jargão técnico sem explicação
- Teoria sem aplicação prática
- Conteúdo passivo (só leitura)
- Sobrecarga de informação

### Proporções Ideais

| Tipo | Proporção |
|------|-----------|
| Conceito/Teoria | 30% |
| Exemplo/Demonstração | 40% |
| Prática/Exercício | 30% |

## Perguntas Essenciais

Antes de criar conteúdo:

1. **Qual o objetivo?** O que o aluno fará depois?
2. **Qual o nível?** Iniciante, intermediário, avançado?
3. **Qual o formato?** Vídeo, texto, apresentação?
4. **Qual a duração?** Tempo disponível?
5. **Qual o contexto?** Como será consumido?
