## Codebase Patterns (KEEP AT TOP)

> **AIOS Principle:** Patterns discovered are shared across all iterations.
> Ralph reads this section FIRST before implementing.

### Architecture Patterns
- Workflows use YAML format with phases containing tasks
- Each task references an agent (from agents/) and optionally a checklist (from checklists/)
- Tasks are markdown files with standard structure
- State is persisted in copy-state.json per project

### Code Conventions
- Workflow files: wf-{number}-{slug}.yaml
- Task files: {action}-{noun}.md (e.g., create-unique-mechanism.md)
- Template files: {purpose}-tmpl.{ext}
- Agents: {name-slug}.md

### Completion Signals
- `<promise>COMPLETE</promise>` - All phases complete
- `<promise>CHECKPOINT:PHASE-X</promise>` - Checkpoint atingido
- `<promise>BLOCKED:T-X.Y</promise>` - Task bloqueado

### What was implemented

- Adicionado suporte a completion signals: COMPLETE, CHECKPOINT:PHASE-X, BLOCKED
- Cria automaticamente: copy-state.json, progress.txt, outputs/
- Detecta workflows disponíveis em workflows/
- Lista projetos ativos com estado atual

### Key Differences from ralph.sh
| Feature | ralph.sh | copy.sh |
|---------|----------|---------|
| State file | prd.json (stories) | copy-state.json (phases) |
| Progress tracking | Stories (passes: true/false) | Phases + tasks |
| Signals | COMPLETE only | COMPLETE, CHECKPOINT, BLOCKED |
| Auto-fix | Verifies output files exist | N/A (phase-based) |
| Output directory | N/A | Creates outputs/ automatically |

### Files Changed
| File | Action | Description |
|------|--------|-------------|
| `expansion-packs/copy/scripts/copy.sh` | Created | Main bash loop (320 lines) |

### Learnings
- **Pattern:** Use `shopt -s nullglob` instead of `2>/dev/null` in for loops
- **Pattern:** copy-state.json tracks phases, tasks, artifacts, checkpoints
- **Pattern:** Environment variables exported for tracking: COPY_ITERATION, COPY_PHASE, COPY_WORKFLOW
