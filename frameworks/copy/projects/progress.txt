## Codebase Patterns (KEEP AT TOP)

> **AIOS Principle:** Patterns discovered are shared across all iterations.
> Ralph reads this section FIRST before implementing.

### Architecture Patterns

- **Agent Structure (Hormozi Standard):** Each copywriter is a markdown file with YAML block containing:
  - `agent:` - Core definition (name, id, title, icon, era, customization)
  - `persona:` - Role, style, identity, focus
  - `core_principles:` - 5-10 guiding principles
  - `commands:` - Agent commands (*framework, *phrase, etc)
  - `operational_frameworks:` - 5-10 detailed frameworks with name, category, origin, definition, principle, steps, application, example, common_mistakes
  - `communication_dna:` - Vocabulary (mandatory/forbidden), rhetorical devices, quick formulas
  - `signature_phrases:` - 30-50 phrases organized by tier (core_mantras, canonical_terminology, executable_formulas)
  - `authority_proof_arsenal:` - Crucible story, authority statistics, proof stack templates
  - `objection_algorithms:` - 3+ algorithms for handling objections

### Research Sources Priority

1. **Minds (if available):** `outputs/minds/{slug}/` - Use analysis/, synthesis/, system_prompts/
2. **WebSearch:** For frameworks, quotes, biography
3. **Context7:** For library documentation if indexed
4. **WebFetch:** For specific pages (archives, blog posts)
5. **NO EXA:** Not configured

### Code Conventions

- Filenames: kebab-case (e.g., `dan-kennedy.md`)
- Copywriter files: `{firstname}-{lastname}.md`
- Agent file location: `expansion-packs/copywriter-os/agents/`

### Testing Patterns

- **Line Count:** No minimum but aim for Hormozi-level depth
- **Activation Test:** Agent must respond with @{agent-name}
- **Voice Consistency:** Output must match copywriter's documented style
- **Hormozi Comparison:** Check all required sections present
